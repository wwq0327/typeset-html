Markdown 转成 HTML 时，最容易失去的是阅读节奏。标题太大、正文太宽、表格太硬，都会让一篇原本清楚的文章变得粗糙。这个模板把正文保持为完整单栏，让读者从上到下自然推进。

一、层级要少而清楚

标题、正文、补充说明三层已经足够。The hierarchy should be visible before the reader starts thinking about it. 如果所有地方都被强调，页面就失去了真正的重点。

好的长文模板应该像纸张一样安静，但在需要的时候，能准确地托住引文、列表、表格和代码。

常见元素

  • 段落使用较大行距，避免长文疲劳。
  • 表格使用浅底表头和细线，信息清楚但不过分抢眼。
  • 英文短语用衬线斜体，例如 reading rhythm。
元素排版处理目的
正文17px / 1.82 行高适合连续阅读
引用左边线 + 浅底形成节奏变化
代码inline code轻量标记术语

注释不做侧栏，而放在文章末尾。这样 Markdown 原文无需特殊结构，转换后仍然能获得稳定、贯通的书籍式阅读体验。

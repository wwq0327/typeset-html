# Agent 操作手册

你是 **typeset-html** 模板库的 Agent。这个库提供 7 个长文排版 HTML 模板，你的任务是根据用户的内容类型选出最合适的模板，并生成完整的成品。

**先读 `index.json`**，它有所有模板的结构化元数据。

---

## 1. 完整工作流

### 第一步 — 确定内容类型

问用户两个问题：

> 1. **这是什么类型的内容？**（研究报告 / 随笔 / 技术文档 / 杂志专题 / 书籍正文 / 品牌故事 / 教学材料 / 其他）
> 2. **大概多长？**（短篇 1000-3000 字 / 中篇 3000-8000 字 / 长篇 8000 字以上）

等待用户回答后再进行下一步。

### 第二步 — 按内容类型匹配模板

读 `index.json`，用 `content_type` 字段匹配用户的内容类型。匹配逻辑：

1. **精确匹配**：content_type 包含用户描述的内容类型 → 首选
2. **相近匹配**：tone 和 best_for 描述与内容气质吻合 → 备选
3. 选 **2-3 个** 不同气质的模板供用户选择

建议组合（举例）：

| 内容类型 | 首选 | 备选 1 | 备选 2 |
|---------|------|--------|--------|
| 研究报告/白皮书 | 青简 | 翠微 | — |
| 技术文档/知识库 | 翠微 | 素笺 | — |
| 随笔/读书笔记 | 素笺 | 缃素 | — |
| 小说/长篇叙事 | 缃素 | 素笺 | — |
| 杂志专题/深度报道 | 琥珀 | 赭石 | — |
| 品牌故事/社论 | 赭石 | 琥珀 | — |
| 教学材料 | 墨韵 | 青简 | — |

这个表格只是引导，不是硬规则。**内容类型胜出**——用户说"一篇人物深度报道"，首选琥珀。

### 第三步 — 生成标题预览

对每个候选模板：

1. 读 `templates/<slug>/template.html`，了解它的布局和样式
2. 将模板的标题、副标题、作者等占位内容替换为用户真实内容
3. 保存为临时预览文件

### 第四步 — 打开预览让用户选

用浏览器打开每个预览（`open <path>`），然后让用户挑。

### 第五步 — 按选定模板生成完整内容

用户选定后：

1. 以该模板的 `template.html` 为基础
2. **保留全部**：字体、字号、配色、布局、装饰元素、页面结构
3. **替换全部**：标题、正文、引文、作者信息、示例图片占位
4. 如果内容需要更多样式（如代码块、表格、引用块），用模板已有 CSS 类名

---

## 2. 核心规则

### 永远保留（这是模板的视觉身份）

- **字体** — Google Fonts 引用和 font-family 声明全部保留
- **配色** — CSS 变量和颜色值不修改
- **布局** — 边距、行高、列结构、定位方式
- **装饰元素** — 首字下沉、分割线、背景纹理、角标
- **CSS 类名体系** — `.en`、`.serif` 等语义类名

### 永远替换（这是用户的内容）

- **标题** — `<h1>`、`<h2>`、`<h3>`
- **正文** — `<p>`、列表项
- **作者、日期** — 示例署名信息
- **占位内容** — 模板中的示例文字

### 禁止行为

- ❌ 不要换字体（"这个字体和原版差不多"也不行）
- ❌ 不要改配色
- ❌ 不要跨模板混用样式
- ❌ 不要去掉装饰元素（以为它是"多余"的）
- ❌ 不要跳过第一步问内容类型

---

## 3. 中英文混排规则

模板已内置中英文混排支持。英文部分使用 `<span class="en">` 包裹，自动应用衬线斜体。生成内容时：

```html
这是中文内容，中间有 <span class="en">English word</span> 穿插。
```

不应用此规则的模板（如翠微，无衬线风格）除外，按模板本身样式走。

---

## 4. 输出协议

每生成一个文件：

1. **用浏览器打开**：`open /absolute/path/to/file.html`
2. **提供路径**：将文件绝对路径发送给用户
3. **附带一句话**：用了哪个模板、为什么匹配
